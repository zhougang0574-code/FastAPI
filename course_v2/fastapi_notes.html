FastAPI 学习笔记 v2 Java 视角

课程结构即学习路径：10 个概念域，简单点合并、难点细拆，全程用 SpringBoot 类比降低门槛。

域 01 · 基础
基础 / 01
Hello FastAPI —— 一次 HTTP 来回的最小闭环
易点·合并 ｜ 对应 SpringBoot：@RestController + 内嵌 Tomcat + Swagger

本课知识点 + SpringBoot 类比

FastAPI作用SpringBoot 对应
FastAPI()创建应用实例@SpringBootApplication 启动类
@app.get/post装饰器注册路由@GetMapping / @PostMapping
返回 dict/list自动序列化 JSON@RestController + Jackson
类型注解参数声明的唯一依据方法形参 + @RequestParam
uvicornASGI 服务器内嵌 Tomcat（异步版）
/docs自带交互文档Swagger UI（此处免配置）

代码

from fastapi import FastAPI

app = FastAPI(title="FastAPI 学习项目", version="0.1.0")

# @app.get("/") 注册为 GET / 的处理器，返回 dict 自动转 JSON
@app.get("/")
def read_root():
    return {"message": "Hello, FastAPI!"}

# item_id: int 这个类型注解 → 自动转换+校验，传 /items/abc 直接 422
@app.get("/items/{item_id}")
def read_item(item_id: int):
    return {"item_id": item_id}

执行流程图

客户端 HTTP 请求 │ ▼ uvicorn（≈ 内嵌 Tomcat）接收 │ ▼ FastAPI 按 (路径 + 方法) 匹配路由 │ ├─ 匹配成功 → 按类型注解解析/校验参数 → 执行函数 → 返回值自动 JSON → 200 │ └─ 参数类型不对 → 422（不进函数体） │ └─ 匹配失败 → 404 启动：cd course_v2/01_基础 && uvicorn 01_hello_fastapi:app --reload 文档：http://127.0.0.1:8000/docs

★ 核心规律

★ FastAPI = Starlette（异步框架）+ Pydantic（数据校验 ≈ Bean Validation）
★ 装饰器即路由注册；返回 dict/list/模型 → 自动 JSON，无需 @ResponseBody
★ 类型注解是理解参数的唯一依据：路径里有同名→路径参数；简单类型没在路径里→查询参数
★ /docs 是免费的交互文档，这是相对 SpringBoot 的最大便利
基础 / 02
async def vs def —— 为什么异步是 FastAPI 的灵魂
难点·细拆 ｜ 无 Java 类比，必须单独搞懂

一句话区别

def 路由
FastAPI 丢进线程池跑。阻塞也没事，≈ SpringBoot 一请求一线程。
async def 路由
在单线程事件循环里跑。靠 await 在 IO 等待时让出，服务别的请求。

四种写法对照

# 1) def：阻塞跑在线程池，安全
@app.get("/sync-sleep")
def sync_sleep():
    time.sleep(1); return {...}

# 2) async def + await 异步IO：高并发的正确姿势
@app.get("/async-sleep")
async def async_sleep():
    await asyncio.sleep(1)   # 让出事件循环
    return {...}

# 3) ✗ 反面教材：async 里写阻塞，霸占循环，拖垮全局并发
@app.get("/async-blocking-BAD")
async def bad():
    time.sleep(1)            # 致命：所有并发请求一起卡住
    return {...}

执行流程图

请求进入 FastAPI │ ┌─────────────┴──────────────┐ ▼ ▼ 路由是 def 路由是 async def │ │ 丢进线程池 在事件循环里跑 （阻塞也没事） ┌─────────┴──────────┐ │ ▼ ▼ ▼ await 异步IO 写了阻塞(time.sleep) 返回响应 （让出，别的请求继续） （霸占循环 ✗ 全卡住）

该用哪个（背下来）

场景选择
用到 await 接口（httpx / 异步 DB 驱动 / asyncio）async def
只有阻塞调用（requests / time.sleep / 同步 SQLAlchemy）普通 def（丢线程池）
纯 CPU 计算（大循环 / 图像处理）普通 def（async 救不了 CPU）
async def 里调用阻塞代码✗ 头号性能事故

★ 核心规律

★ def → 线程池（Java 直觉，阻塞安全）；async def → 事件循环（高并发但禁止阻塞）
★ 高并发来源是 await：IO 等待时让出循环去服务别人，而不是堆线程
★ 头号事故：async def 里调阻塞代码（time.sleep/requests/同步 DB），拖垮全局
★ 不确定就用普通 def——FastAPI 安全地丢线程池，绝不会比写错的 async 更糟
域 02 · 请求参数
请求参数 / 01
路径参数 + 查询参数
易点·合并 ｜ ≈ @PathVariable + @RequestParam

知识点 + 类比

FastAPI规则SpringBoot
路径里有同名占位路径参数@PathVariable
简单类型没在路径里查询参数@RequestParam
有默认值 / 无默认值可选 / 必填(422)required=false / true

def orders(user_id: int, status: Optional[str] = None, page: int = 1):

执行流程图

参数进阶：Enum / 路由顺序 / Query 约束
进阶·合并

知识点

Enum 限值
路径/查询参数限定合法取值，/docs 自动下拉
路由顺序陷阱

ge/le/min_length/max_length + description 文档

... 表示必填；给真实默认值则可选

执行流程图

★ 核心规律

★ 路由顺序陷阱：固定路径写在变量路径之前（不像 Spring 按精确度匹配）
★ Query(默认, ge/le/min_length, description) = 校验 + 文档一次写全
域 03 · 请求体与 Pydantic
请求体与Pydantic / 01
请求体 + Field 约束 + 嵌套模型
易点·合并 ｜ ≈ DTO + Bean Validation

知识点

BaseModel 做参数
自动从 body 读 JSON（≈ @RequestBody），无需注解
Field()
gt/le/min_length + description（≈ @Min/@Size + Swagger）
嵌套模型
字段类型是另一个 BaseModel，递归校验
三类共存
路径/查询/请求体同函数，按来源自动分流
class UserCreate(BaseModel):
    name: str = Field(min_length=1, max_length=50)
    age: int = Field(gt=0, le=150)
    address: Optional[Address] = None   # 嵌套模型

@app.post("/users")
def create(user: UserCreate): ...   # ≈ @RequestBody @Valid

执行流程图

POST body JSON → Pydantic 校验 age=200 → 不满足 le=150 → 422（指明字段） 全合法 → 反序列化成对象进函数

★ 核心规律

★ BaseModel 子类做参数 → 自动当请求体；Field ≈ Bean Validation + Swagger 一处写全
★ 嵌套模型 = DTO 套 DTO，body 就是嵌套 JSON，自动递归校验
请求体与Pydantic / 02
表单 Form + 文件上传 UploadFile
≈ MultipartFile

知识点

方式用途
Form(...)表单字段（非 JSON），OAuth2 登录就用它
bytes = File(...)小文件，一次读进内存
UploadFile大文件，流式省内存，有 .filename/.content_type（推荐）
@app.post("/upload")
async def upload(file: UploadFile):
    content = await file.read()
    return {"filename": file.filename, "size": len(content)}

执行流程图

multipart/form-data ├─ 文本字段 → Form(...) └─ 文件 → UploadFile（流式）/ bytes+File（进内存） ★ 用 Form/File 必须装 python-multipart，否则启动报错

★ 核心规律

★ 小文件 bytes=File(...)；大文件 UploadFile（推荐，流式，await read()）
★ 一个请求里有 Form 就整体按表单解析，不能再混 JSON body
请求体与Pydantic / 03
自定义校验 field_validator / model_validator
进阶

知识点

field_validator
单字段自定义校验/转换（≈ 自定义 ConstraintValidator）
model_validator(after)
跨字段校验，如两次密码一致
@field_validator("password")
def strength(cls, v):
    if len(v) < 6: raise ValueError("至少6位")  # → 422
    return v

@model_validator(mode="after")
def match(self):
    if self.password != self.password2: raise ValueError("不一致")

执行流程图

校验顺序：Field 约束 → field_validator(逐字段,可改值) → model_validator(跨字段) 任一步 raise ValueError → 422

★ 核心规律

★ 能用 Field 表达的就别写校验器；跨字段校验用 model_validator(after)
★ 校验失败统一 raise ValueError，FastAPI 自动包成 422（Pydantic v2 写法）
域 04 · 响应
响应 / 01
response_model + 状态码 status_code
易点·合并 ｜ ≈ 返回 DTO + @ResponseStatus

知识点

response_model
出参契约，自动过滤多余字段（密码不在模子→丢弃）
XxxIn / XxxOut
分离请求/响应模型，密码只进不出
status_code
201 Created / 204 No Content
exclude_unset
只返回赋过值的字段，适合 PATCH
@app.post("/users", response_model=UserOut, status_code=201)
def create(user: UserIn):
    return user   # 即便含 password，UserOut 没这字段 → 不泄露

执行流程图

函数 return 对象(字段多,含敏感) → response_model 当出参模子过滤(只留声明字段) → exclude_unset 再丢未赋值字段 → 带 status_code 序列化返回

★ 核心规律

★ response_model 是"出参契约"：函数返回多余字段会被自动过滤，天然防泄露
★ 输入 XxxIn、输出 XxxOut 分离；密码只进不出
响应 / 02
自定义 Response / 多形态 / 流式
进阶·合并

知识点

JSONResponse
自定义状态码/响应头（≈ ResponseEntity）
Union[A,B]
允许返回多种结构
StreamingResponse
边 yield 边发，大文件/SSE/AI 打字机
async def gen():
    for i in range(5):
        yield f"data: {i}\n\n"
        await asyncio.sleep(0.3)   # 异步，不阻塞循环
return StreamingResponse(gen(), media_type="text/event-stream")

执行流程图

普通 dict/模型 → 默认 JSONResponse 要控状态码/头 → 直接返回 JSONResponse 要边算边发 → StreamingResponse(生成器) → 客户端逐块收

★ 核心规律

★ 要控状态码/响应头就直接返回 JSONResponse；多结构用 Union[A,B]
★ StreamingResponse 接生成器，流式里用 async + await，别用阻塞 sleep
域 05 · 错误处理
错误处理 / 01
HTTPException + 自定义处理器 + 校验错误
易点·合并 ｜ ≈ @ControllerAdvice

知识点

机制用途SpringBoot
HTTPException就地抛标准错误(404/400)ResponseStatusException

依赖注入 / 02
yield 依赖：资源的打开→使用→关闭
难点·细拆 ｜ 无 Java 直接类比

知识点

yield 前 / 后
前=准备资源，后=释放资源
收尾时机
yield 后的代码在"响应返回之后"执行
try/finally
路由抛异常也保证释放（≈ try-with-resources）
def get_session():
    session = FakeSession()
    try:
        yield session     # 注入给路由，函数在此暂停
    finally:
        session.close()   # 响应返回后从这里恢复，关资源

执行流程图

请求 → get_session 执行到 yield(打开会话,交给路由) → 路由用 session → 返回响应 → 响应发出后从 yield 恢复 → finally 关闭会话 抛异常时：路由 raise → finally 仍关闭 → 再向上抛

★ 核心规律

★ yield 依赖 = 资源生命周期管理：yield 前准备、yield 后释放（≈ try-with-resources）
★ 这就是数据库 get_db 的标准范式，〔09/01〕直接用它
依赖注入 / 03
路由组依赖 / 全局依赖
进阶 ｜ ≈ 拦截器

知识点

层级范围SpringBoot

★ 核心规律

★ dependencies=[...] 里的依赖"只执行不注入值"，适合鉴权/限流/审计
★ 三层级：FastAPI 全局 / APIRouter 路由组 / 单路由（≈ 拦截器范围）
域 07 · 中间件与后台
中间件与后台 / 01
中间件 + CORS
易点·合并 ｜ ≈ Filter / CorsFilter

知识点

@app.middleware("http")
拦截所有请求/响应（≈ Filter）
await call_next
放行到下一层（≈ chain.doFilter）
洋葱模型
请求正序进、响应逆序出
CORSMiddleware
前后端分离必配（≈ @CrossOrigin）
@app.middleware("http")
async def timing(request, call_next):
    start = time.time()
    response = await call_next(request)   # 放行
    response.headers["X-Process-Time"] = ...
    return response

执行流程图

请求 → CORS → timing → 路由 响应 ← CORS ← timing ← 路由 （洋葱：进正序、出逆序）

★ 核心规律

★ 中间件拦截所有请求/响应（≈ Filter），await call_next 放行；后加的在更外层
★ CORSMiddleware 前后端分离必配；中间件是异步的，别写阻塞代码
中间件与后台 / 02
后台任务 BackgroundTasks
≈ @Async

知识点

add_task(fn, args)
登记"响应返回后"执行的活
适用
发邮件/写日志/清理，客户端不等结果
边界
同进程非分布式；重活该上 Celery
@app.post("/register")
def register(email: str, tasks: BackgroundTasks):
    tasks.add_task(send_email, email, "欢迎")  # 登记，不执行
    return {"msg": "注册成功"}   # 立刻返回，邮件后台发

执行流程图

核心逻辑 → add_task(只登记) → 返回响应(客户端拿到结果) →(响应发出后) 后台依次执行已登记任务

★ 核心规律

★ add_task 登记"响应后执行"的轻量活（≈ @Async）
★ 要重试/持久化的重活该上 Celery/RQ，BackgroundTasks 不替代它
域 08 · 安全认证（整域细拆，无 Java 类比）
安全认证 / 01
OAuth2 骨架：登录端点 + 受保护路由
难点·细拆

知识点

OAuth2PasswordBearer
声明 token 从哪拿，驱动 /docs 的 Authorize
OAuth2PasswordRequestForm
标准登录表单 username/password
受保护路由
注入 oauth2_scheme，缺 token → 401
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")
@app.get("/me")
def me(token: Annotated[str, Depends(oauth2_scheme)]):
