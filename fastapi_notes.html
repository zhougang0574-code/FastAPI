第 1 课
Hello FastAPI —— 快速开始

什么是 FastAPI

FastAPI 是一个基于 Python 类型提示 构建的现代 Web 框架， 性能与 Node.js / Go 相当，同时极大减少了开发错误。

它的三个核心优势：

★速度
基于 Starlette（ASGI 异步框架），性能极高
★少写代码
类型提示即校验，少写约 40% 样板代码
★自动文档
/docs 和 /redoc 开箱即用，零配置

FastAPI 的依赖关系：

FastAPI ├── Starlette ← 处理 HTTP / WebSocket / 异步 └── Pydantic ← 数据校验 / 序列化 / 类型转换

第一个 FastAPI 应用

安装：

# pip install "fastapi[standard]" 会同时安装 uvicorn
pip install "fastapi[standard]"

代码（01_hello_fastapi.py）：

from fastapi import FastAPI

# 创建应用实例，title/description/version 显示在 /docs
app = FastAPI(
    title="FastAPI 学习项目",
    description="从零学习 FastAPI",
    version="0.1.0",
)

# @app.get("/") 把函数注册为 GET "/" 的处理器
# 返回 dict → 自动变成 JSON 响应
@app.get("/")
def read_root():
    return {"message": "Hello, FastAPI!"}

# {item_id} 声明路径参数，类型提示 int 触发自动校验
@app.get("/items/{item_id}")
def read_item(item_id: int):
    return {"item_id": item_id}

@app.post("/items")
def create_item(name: str, price: float):
    return {"name": name, "price": price, "status": "created"}

启动命令：

# 格式：uvicorn 文件名:app实例名 --reload
uvicorn 01_hello_fastapi:app --reload

# 输出示例：
# INFO:     Uvicorn running on http://127.0.0.1:8000
# INFO:     Waiting for application startup.
# INFO:     Application startup complete.
提示： --reload 是开发模式，代码修改后自动重启。 生产环境部署时去掉这个参数。

执行流程图

客户端发送 HTTP 请求 │ ▼ uvicorn（ASGI 服务器）接收请求 │ ▼ FastAPI 路由匹配（path + method） │ ├─ 匹配成功 │ │ │ ▼ │ 类型校验（由类型提示自动完成） │ │ │ ▼ │ 执行路径操作函数 │ │ │ ▼ │ 返回值序列化为 JSON → 200 OK │ └─ 匹配失败 → 404 Not Found 类型校验失败 → 422 Unprocessable Entity

自动文档

服务启动后，以下地址开箱即用，无需任何配置：

第 2 课
路径参数 Path Parameters

路径参数基础

用 {param} 在路径里声明参数，函数签名里同名接收，类型提示自动完成转换和校验。

@app.get("/users/{user_id}")
def get_user(user_id: int):
    return {"user_id": user_id}

# GET /users/42   → {"user_id": 42}   ✓
# GET /users/abc  → 422 自动报错     ✓

路由顺序陷阱

陷阱：固定路径必须注册在变量路径之前，否则固定值会被当成变量参数！
# ✓ 正确：固定路径写前面
@app.get("/users/me")
def get_current_user(): ...

@app.get("/users/{user_id}")
def get_user(user_id: int): ...

Enum 限制合法值

from enum import Enum

class ModelName(str, Enum):
    alexnet = "alexnet"
    resnet  = "resnet"
    lenet   = "lenet"

@app.get("/models/{model_name}")
def get_model(model_name: ModelName):
    return {"model": model_name.value}

# GET /models/vgg → 422（不在 Enum 里）

path 类型：参数含斜杠

# {file_path:path} 让参数可以包含 /
@app.get("/files/{file_path:path}")
def read_file(file_path: str):
    return {"path": file_path}

# GET /files/home/user/a.txt → {"path":"home/user/a.txt"}

API 速查

写法说明
{item_id}声明路径参数
item_id: int自动类型校验转换
class X(str, Enum)限制合法值
{path:path}参数可含斜杠
固定路由写前面避免被变量路由吃掉
★类型提示是运行时校验，不只是注释
★Enum 参数：合法值即文档，非法值即错误
★注册顺序 = 匹配优先级
第 3 课
查询参数 Query Parameters

查询参数基础

函数参数不在路径里 → 自动成为查询参数（?key=value）

@app.get("/items")
def list_items(skip: int = 0, limit: int = 10):
    return items[skip: skip + limit]

# GET /items           → skip=0, limit=10
# GET /items?skip=20   → skip=20, limit=10

可选参数与必填参数

from typing import Optional

@app.get("/items/{item_id}")
def get_item(item_id: int, q: Optional[str] = None):
    # q 可选：传 ?q=hello 则有值，不传则 None
    return {"id": item_id, "q": q}

@app.get("/search")
def search(keyword: str):  # 无默认值 = 必填
    return {"keyword": keyword}
# GET /search            → 422（keyword 必填）
# GET /search?keyword=hi → 正常

bool 类型自动转换

@app.get("/items/{item_id}")
def get_item(item_id: int, short: bool = False):
    return item_id

# ?short=1     → True
# ?short=true  → True
# ?short=on    → True
# ?short=yes   → True
# ?short=false → False（大小写不敏感）

Query() 添加约束

from typing import Annotated
from fastapi import Query

@app.get("/search")
def search(
    q: Annotated[Optional[str], Query(min_length=2, max_length=50)] = None,
    page: Annotated[int, Query(ge=1, description="页码")] = 1,
):
    return {"q": q, "page": page}

API 速查

写法说明
param: int = 0可选参数，默认值 0
param: str必填参数（无默认值）
Optional[str] = None可为 None 的可选参数
param: bool = Falsebool 自动转换 1/true/on/yes
Query(ge=1, le=100)添加范围约束
★区分三种参数：路径参数 / 查询参数 / 请求体
★bool 转换不区分大小写，1/true/on/yes 都是 True
★Query() 约束写在 Annotated 里，类型与约束分离更清晰
第 4 课
请求体 + Pydantic 数据模型

BaseModel 定义请求体

继承 BaseModel 定义结构，函数参数类型是 BaseModel 子类时，FastAPI 自动从 request body 读取。

from pydantic import BaseModel
from typing import Optional

class Item(BaseModel):
    name: str            # 必填
    price: float         # 必填
    tax: Optional[float] = None   # 可选

@app.post("/items")
def create_item(item: Item):
    return item.model_dump()
请求时需设置 Content-Type: application/json，body 是 JSON 格式。

Field() 添加验证约束

from pydantic import BaseModel, Field

class Item(BaseModel):
    name: str = Field(min_length=1, max_length=100)
    price: float = Field(gt=0, description="价格大于0")
    stock: int = Field(ge=0, default=0)

# gt  = greater than（严格大于）
# ge  = greater than or equal（大于等于）
# lt  = less than / le = less than or equal

嵌套模型

class Image(BaseModel):
    url: str
    name: str

class Item(BaseModel):
    name: str
    image: Optional[Image] = None   # 嵌套另一个模型
    tags: list[str] = []            # 列表字段

# 请求体：{"name":"Phone","image":{"url":"...","name":"front"}}

三种参数同时使用

FastAPI 区分参数来源的规则： 路径里有 {xxx} → 路径参数 参数类型是 BaseModel → 请求体（从 body 读） 其他 → 查询参数（从 URL ?key=val 读） @app.put("/items/{item_id}") def update(item_id: int, ← 路径参数 item: Item, ← 请求体 q: str = None): ← 查询参数

Field(gt=0)添加数值约束

第 5 课
响应模型 Response Model

过滤敏感字段

response_model 参数控制返回给客户端的字段，即使函数返回了多余的字段也会被过滤。

class UserIn(BaseModel):
    username: str;  password: str;  email: str

class UserOut(BaseModel):
    username: str;  email: str   # 不含 password

# response_model=UserOut 确保 password 不出现在响应里
@app.post("/users", response_model=UserOut, status_code=201)
def create_user(user: UserIn):
    return user   # 含 password，但客户端看不到

response_model_exclude_unset

class Item(BaseModel):
    name: str;  price: float
    tax: float = 10.5   # 有默认值

@app.get("/items/{id}",
         response_model=Item,
         response_model_exclude_unset=True)
def get_item(id: int):
    return Item(name="Phone", price=999.0)
    # 响应：{"name":"Phone","price":999.0}
    # tax 没有赋值 → 不返回（即使有默认值）
适用场景：PATCH 接口响应，避免默认值"污染"返回值。

响应状态码

from fastapi import status

@app.post("/items", status_code=status.HTTP_201_CREATED)
def create(): ...

@app.delete("/items/{id}", status_code=status.HTTP_204_NO_CONTENT)
def delete(): ...   # 204 不能有 response body
200 OK — 默认
201 Created — 创建成功
204 No Content — 删除成功
422 Unprocessable — 校验失败

第 6 课
错误处理 HTTPException

HTTPException

from fastapi import HTTPException, status

@app.get("/items/{item_id}")
def read_item(item_id: str):
    if item_id not in db:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=f"Item '{item_id}' not found",
            headers={"X-Error": "item-not-found"},
        )
    return db[item_id]

# detail 也可以是 dict（更丰富的错误信息）
raise HTTPException(status_code=400, detail={
    "code": "INVALID_INPUT",
    "field": "price",
    "message": "Price must be positive"
})

自定义异常处理器

class AppError(Exception):
    def __init__(self, code: str, msg: str):
        self.code = code; self.msg = msg

@app.exception_handler(AppError)
async def app_error_handler(request, exc: AppError):
    return JSONResponse(
        status_code=400,
        content={"code": exc.code, "message": exc.msg},
    )

@app.get("/test")

覆盖默认 422 格式

from fastapi.exceptions import RequestValidationError

@app.exception_handler(RequestValidationError)
async def validation_handler(request, exc):
    return JSONResponse(
        status_code=422,
        content={
            "code": "VALIDATION_ERROR",
            "errors": exc.errors(),
        },
    )

第 7 课
依赖注入 Dependency Injection

基础依赖

把公共逻辑抽成依赖函数，多个路由通过 Depends() 复用，避免重复代码。

from fastapi import Depends
from typing import Annotated

def common_params(skip: int = 0, limit: int = 100):
    return {"skip": skip, "limit": limit}

CommonDeps = Annotated[dict, Depends(common_params)]

@app.get("/items")
def list_items(commons: CommonDeps): return commons

@app.get("/users")
def list_users(commons: CommonDeps): return commons

类作为依赖

class QueryFilter:
    def __init__(self, q: Optional[str] = None, skip: int = 0):
        self.q = q
        self.skip = skip

@app.get("/items")
def list_items(f: Annotated[QueryFilter, Depends(QueryFilter)]):
    return {"q": f.q, "skip": f.skip}

嵌套依赖 & 路由级依赖

def verify_key(x_key: Annotated[str, Header()]):
    if x_key != "secret": raise HTTPException(400)

# dependencies=[] 只执行副作用，不注入返回值
@app.get("/secure", dependencies=[Depends(verify_key)])
def secure(): return {"ok": True}

yield 依赖：资源生命周期

def get_db():
    db = SessionLocal()   # setup：打开连接
    try:
        yield db           # 注入路由函数
    finally:
        db.close()         # teardown：请求结束关闭

@app.get("/items")
def items(db: Annotated[Session, Depends(get_db)]):
    return db.query(Item).all()
请求开始 → get_db() 执行到 yield → 路由函数执行 → 响应发出 → finally db.close() 清理

API 速查

写法说明
Depends(func)声明依赖，注入返回值
Annotated[T, Depends(X)]推荐写法，类型与依赖合并
dependencies=[Depends(X)]路由级副作用依赖
yield 依赖管理需要清理的资源
★依赖注入是解耦公共逻辑的核心手段
★相同依赖在一次请求内只执行一次（自动缓存）
★yield 依赖是数据库会话管理的标准模式
第 8 课
中间件与 CORS

CORSMiddleware：解决跨域

CORS（跨源资源共享）：浏览器安全策略，前端域名与 API 域名不同时会被拦截。

from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],          # 生产：["https://yourdomain.com"]
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
生产环境把 "*" 改为具体域名，避免安全风险。

自定义中间件

import time

@app.middleware("http")
async def add_process_time(request, call_next):
    start = time.perf_counter()
    response = await call_next(request)  # 必须 await！
    elapsed = time.perf_counter() - start
    response.headers["X-Process-Time"] = f"{elapsed*1000:.2f}ms"
    return response
中间件是 AOP（面向切面编程）思想：日志、认证、限流都适合放在这里，与业务逻辑解耦。

洋葱模型：执行顺序

最后注册的中间件最先执行（包裹最外层） 请求 → MW-C → MW-B → MW-A → 路由函数 响应 ← MW-C ← MW-B ← MW-A ←（响应往回走） add_middleware(A) ← 最先注册，最后执行 add_middleware(B) add_middleware(C) ← 最后注册，最先执行

第 9 课
安全认证 OAuth2 + JWT

JWT 原理

JWT 结构：Header.Payload.Signature（Base64 编码） Header: {"alg":"HS256","typ":"JWT"} Payload: {"sub":"alice","exp":1234567890} Signature: HMACSHA256(header+payload, SECRET_KEY) 特点： ✓ 无状态：服务器不存 Session，靠签名验证 ✓ 自包含：payload 里携带用户信息 ✗ 无法撤销：token 未过期就一直有效（需配合黑名单）

密码哈希存储

from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# 注册时哈希密码存入数据库
hashed = pwd_context.hash("mysecret")

# 登录时验证（无法反推原始密码）
is_ok = pwd_context.verify("mysecret", hashed)  # True
永远不要存明文密码！即使数据库泄露，bcrypt 哈希也无法反推原始密码。

Token 生成与验证

from jose import jwt, JWTError
from fastapi.security import OAuth2PasswordBearer

SECRET_KEY = "your-secret"  # 生产用 secrets.token_hex(32)
ALGORITHM = "HS256"

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/token")

def create_token(data: dict) -> str:
    payload = {**data, "exp": datetime.now() + timedelta(minutes=30)}
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)

# 登录接口：验证账密 → 返回 JWT
@app.post("/token")
async def login(form: Annotated[OAuth2PasswordRequestForm, Depends()]):
    user = authenticate(form.username, form.password)
    token = create_token({"sub": user["username"]})
    return {"access_token": token, "token_type": "bearer"}

保护路由

async def get_current_user(token: Annotated[str, Depends(oauth2_scheme)]):
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        username = payload.get("sub")
    except JWTError:
        raise HTTPException(401, headers={"WWW-Authenticate": "Bearer"})
    return get_user(username)

# 只需注入依赖，路由自动受保护
@app.get("/me")
async def me(user: Annotated[dict, Depends(get_current_user)]):
    return user

ORM 模型 vs Pydantic Schema

from sqlalchemy.orm import DeclarativeBase
from sqlalchemy import Column, Integer, String, Float

class Base(DeclarativeBase): pass

# ORM 模型：对应数据库表
class ItemModel(Base):
    __tablename__ = "items"
    id = Column(Integer, primary_key=True)
    name = Column(String, nullable=False)
    price = Column(Float)

# Pydantic Schema：对应 API 输入/输出
class ItemResponse(BaseModel):
    id: int; name: str; price: float
    class Config: from_attributes = True  # 允许从 ORM 对象构建
ORM Model → 数据库操作（增删改查） Pydantic → API 层（序列化、校验、文档） 两者分离：一个模型只做一件事

会话管理：yield 依赖

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

@app.post("/items", response_model=ItemResponse, status_code=201)
def create(item: ItemCreate, db: Session = Depends(get_db)):
    db_item = ItemModel(**item.model_dump())
    db.add(db_item)
    db.commit()
    db.refresh(db_item)  # 拿到自增 id
    return db_item

# PATCH 专用 Schema：所有字段 Optional
class ProductUpdate(BaseModel):
    name: Optional[str] = None
    price: Optional[float] = None

@app.patch("/products/{id}")
def patch(id: int, data: ProductUpdate, db=Depends(get_db)):
    product = get_or_404(id, db)
    # exclude_unset=True：只拿客户端实际传的字段
    for k, v in data.model_dump(exclude_unset=True).items():
        setattr(product, k, v)
    db.commit(); db.refresh(product)
    return product

# PATCH {"price":99} → 只改 price，name 保持原值
# PUT  {"name":"A","price":99} → 必须传所有字段，全量替换

N+1 问题 & joinedload 解法

from sqlalchemy.orm import joinedload

# 有 N+1 问题（查 10 个 user 会触发 10 条 posts 查询）
users = db.query(User).all()

# 用 joinedload：一次 JOIN 查询拿所有数据
users = (
    db.query(User)
    .options(joinedload(User.posts).joinedload(Post.tags))
    .all()
)
N+1 问题： 查 User 列表 → 1 条 SQL for user in users: 访问 user.posts → 每个 user 触发 1 条 SQL → 1 + N 条 SQL（N=用户数） joinedload 解法： 1 条 SQL（LEFT JOIN）拿所有数据

# 1. Annotated 简化重复依赖声明
DbDep = Annotated[Session, Depends(get_db)]

# 2. 过滤 + 分页 + joinedload 一次搞定
@app.get("/tasks")
def list_tasks(status: Optional[TaskStatus]=None,
               tag_name: Optional[str]=None,
               skip: int=Query(0,ge=0),
               limit: int=Query(10,ge=1,le=100),
               db: DbDep = None):
    query = (db.query(TaskModel)
             .options(joinedload(TaskModel.tags))
             .filter(TaskModel.is_deleted == False))
    if status: query = query.filter(TaskModel.status == status.value)
    if tag_name: query = query.join(TaskModel.tags).filter(TagModel.name == tag_name)
    return query.offset(skip).limit(limit).all()

# 3. 统计接口
@app.get("/stats")
def stats(db: DbDep):
    active = TaskModel.is_deleted == False
    return {s.value: db.query(TaskModel).filter(active,
        TaskModel.status==s.value).count() for s in TaskStatus}
